# f2l(Fetch to Local)

## Need

### Fetch problems from remote

- Different platforms use different webpage paths as parameters for the tool
- Fetch problems
- Fetch test cases
- Fetch test frameworks

### Generate a corresponding folder locally for each questions

```txt
test/
  |-- question.md Question
  |-- solution.cpp Code framework
```

### Work on Local

- Code in `solution.cpp`
- Compile and run test cases locally
- Local judge

### Push to remote

- After completing the code locally, you can push it to the corresppnding platform
- Waiting for the OJ results to be returned

## Don't Need

- Fetch pages other than the questing
- Fetch comments and solutions for the questions
